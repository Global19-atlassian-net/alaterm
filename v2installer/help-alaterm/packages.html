3.1. Updating Alaterm itself

When the Alaterm installer finishes, it is done. You can discard the files from the Alaterm repository at GitHub.

If there is an update to Alaterm, it will only involve some change to one or more configuration files. It will not re-install. Launch Alaterm, then type:

echo $scriptVersion

The Alaterm repository will have a folder named update. Compare your script version to the LATEST version. There will be two LATEST versions. One pertains to version 1.x, and its number will be 0-199. The other pertains to version 2.x, and its number will be 200-299. If your scriptVersion is less than the LATEST, then bash update.bash applies the update. Note that version 1.x cannot be updated to version 2.x. They are different.

3.2. Using pacman for Arch software

Programs, such as GIMP or LibreOffice, come from the Arch Linux ARM project. This is a subset of Arch Linux. Within that subset, some programs will work in your system, but others will not.

Use pacman (not to be confused with the game) to install, remove, and update programs. This must be done from the Terminal. There is no GUI Menu choice.

Some popular commands, case-sensitive:

pacman -Syu # updates all installed.

pacman -S program # installs program

pacman -R program # removes progam

For more information, look here.

Arch Linux does not have separate "dev" packages. When a program is installed, dev headers are also installed.

Removing a program does not remove any dependencies installed at the same time. For your convenience, command autoremove will clean up unnecessary packages.
